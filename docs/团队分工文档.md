# 团队分工文档

## 1. 分工原则

本项目按“数据预处理 → EV建模 → EMS调度 → 分解校验 → 拓展分析 → 报告整合”的流程推进。

总体原则：

1. A 负责数据底座、无序充电基准、结果校验和报告整合；
2. B 负责 EV 集群建模和单车分解；
3. C 负责 EMS 系统调度和渗透率分析；
4. D 负责价格机制拓展；
5. 各成员负责自己模块的基础可视化；
6. A 统一图表风格、文件格式和最终报告。

## 7. B 与 C 的协作关系

B 和 C 在模型逻辑上耦合，但代码实现上通过文件接口解耦。

```text
B 输出 EV 聚合约束
        ↓
C 读取 EV 聚合约束并求解 EMS 调度
        ↓
C 输出 EV 集群总充放电功率
        ↓
B 读取调度结果并分解到单车
```

对应文件接口：

```text
B → C:
data/processed/ev_agg_bounds_ordered_56.csv
data/processed/ev_agg_bounds_v2g_56.csv

C → B:
results/dispatch/dispatch_ordered.csv
results/dispatch/dispatch_v2g.csv
```

## 9. 统一变量与单位

### 时间

```text
主任务：
t = 0, 1, ..., 55
对应 18:00–8:00，共 56 个 15 min 时段。

拓展任务：
t_day = 0, 1, ..., 95
对应 00:00–24:00，共 96 个 15 min 时段。

时间步长：
dt_h = 0.25
```

### 单位

| 类型        | 单位 |
| ----------- | ---- |
| 单车功率    | kW   |
| 单车电量    | kWh  |
| EV 集群功率 | MW   |
| EV 集群电量 | MWh  |
| 系统功率    | MW   |
| 系统电量    | MWh  |

### EV 净功率

```text
p_ev_net_mw = p_ev_ch_mw - p_ev_dis_mw
```

含义：

```text
p_ev_net_mw > 0：EV 集群整体充电
p_ev_net_mw < 0：EV 集群整体放电
```

### 场景名称

```text
unordered：无序充电
ordered：有序充电
v2g：有序充放电
```

## 11. 提交规范

1. 不修改 `data/raw/` 中的原始数据；
2. 所有模块通过 CSV、NPZ 或 JSON 文件交互；
3. 不允许直接调用他人脚本中的中间变量；
4. 重要参数写入 `config.yaml`；
5. 变量名统一使用英文和下划线；
6. 单车层用 kW/kWh，集群和系统层用 MW/MWh；
7. 每个模块运行后必须保存标准输出文件；
8. 图表保存到 `results/figures/`；
9. 表格保存到 `results/tables/`。

# report_C_1.md

## 任务摘要
在组员A完成数据预处理和无序充电基准、组员B完成EV集群聚合建模（阶段一）的基础上，我（成员C）主要完成了EMS调度优化模型（支持无序、有序和V2G三种场景）的求解，以及不同EV渗透率对系统运行成本和弃风影响的敏感性分析。

## 详细工作内容

1. **EMS 优化调度建模 (`src/models/03_ems_dispatch.py`)**：
   - 使用 `cvxpy` 构建了基于 56 个时段（15分钟级）的优化模型。
   - **决策变量**：各热电机组的出力曲线、风电消纳功率、EV 充放电功率（仅有序和 V2G 场景）。
   - **目标函数**：最小化系统总运行成本（Total Operational Cost）。不仅包含热电机组的综合运行成本（二次项、一次项和常数项），还显式加入了 EV 充放电的结算费用（即支付给用户的放电费减去向用户收取的充电费）。
   - **算法改进与 big-M 方法说明**：结合模型的经济学本质，由于放电补偿电价高于充电电价且存在能量损耗，系统同时充放电会导致总运行成本严格增加。因此，优化模型天然满足互补松弛条件（$P_{ch} \cdot P_{dis} = 0$）。为了避免引入 0-1 整数变量将问题升级为极难求解的混合整数二次规划（MIQP），本程序摒弃了冗余的 big-M 整数约束，采用高效的连续凸二次规划进行求解（详见单独的方法学说明文档 `report_C_method.md`）。
   - **约束条件**：
     - 系统功率平衡约束。
     - 热电机组出力上下限约束及相邻时刻的爬坡率限制。
     - 风电消纳上限约束（不超过预测值）。
     - EV 集群充放电功率上下限约束（由 B 提供）。
     - EV 集群累积能量边界约束（由 B 提供），并要求最后一个时段必须满足总充电量需求。

2. **EV 渗透率敏感性分析 (`src/models/04_penetration_sensitivity.py`)**：
   - 编写了循环脚本，评估了渗透率从 0% 倍至 200% 倍规模下的系统运行结果。
   - 提取了每次求解的综合运行成本、总弃风量、总消纳风量等关键指标，并输出至 `results/sensitivity/penetration_sensitivity.csv`。

3. **结果可视化 (`src/analysis/05_plot_dispatch_results.py`)**：
   - 生成了各场景下的 EV 净充电功率对比图、弃风情况对比图。
   - 绘制了 V2G 场景下的系统负荷、风电消纳及各热电机组出力的堆叠图。
   - 绘制了渗透率敏感性分析图表（运行成本随渗透率变化、弃风量随渗透率变化等），所有图表输出到 `results/figures/` 目录下。

4. **项目维护与文档同步**：
   - 更新了 `README.md` 中关于组员 C 的当前代码定位，并增补了关于爬坡初值、最终充电量强制约束的统一优化前提假设。
   - 修改了 `run_all.py` 的执行流，将其无缝对接到整个自动化工作流中。
